# Portfolio 项目工作规则

本项目是自动化作品集整理流程。后续在 `/Users/Sue/Documents/Portfolio` 中开启新对话时，优先按这里的规则协助用户。

## 固定目标

- 以 `portfolio_registry.xlsx` 作为唯一作品台账。
- 新媒体作品：用户提供发布链接和基础信息后，运行入库流程提取正文并归档。
- 纸刊作品：用户把文档放入 `inbox/`，并在台账填写文件名和基础信息后，运行入库流程归档。
- 输出包括两类：静态网站 `public/` 和定制 PDF `exports/pdf/`。

## 台账格式规则

- `作品台账` 第一行冻结，便于滚动查看字段名。
- `来源类型`、`撰稿类型`、`是否公开展示`、`是否为精选作品`、`处理状态` 必须设置为下拉列表。
- 表格边框只使用横线：正文行使用灰色横线，第一行使用上下黑色边框，不设置竖线和表格尾边框。

## 网站生成规则

- 网站首页标题固定为 `Su FAN 作品集`。
- 首页个人简介固定展示：
  - 现为自由撰稿人，base 北京。
  - 中英文流利，能从容应对双语采访与撰稿任务。
  - 教育背景：经济学博士，北京大学（2025）；联合培养博士研究生，巴黎第一大学（2023）；经济学、社会学学士，北京大学（2020）。
- 首页需要从已入库作品的 `发布媒体` 字段自动汇总并列出合作媒体名称。
- 首页个人照片固定读取 `assets/profile.jpg`，不进入台账。
- 首页个人照片放在合作媒体右侧，尺寸接近精选作品头图，并固定对齐到首页资料区最右栏。
- 台账只保留可选字段 `作品图片`。作品图片建议放入项目 `assets/` 文件夹，并在台账填写如 `20260629-男人风尚.jpg` 的文件名，或填写完整本地路径/图片 URL。
- 如果填写了 `作品图片`，精选作品右侧、作品总览左侧、单篇文章页正文栏左边展示对应图片。未填写图片时不保留空白占位。
- 如果作品的 `链接/文件名` 是网址，网页端在作品标题下方显示 `阅读原文` 超链接，指向原始发布地址；纸刊文件名不生成该链接。
- 作品图片以竖版为主，竖屏浏览时也保持竖版图片框。
- 网站图片保持彩色显示，不使用黑白滤镜；图片主体完整显示，背景使用同图模糊铺底来保持版面整齐，不裁切关键信息。
- 图片主体边缘使用淡出过渡，模糊背景也要有边缘柔化，使主图自然衔接到铺底背景。
- 当前网站风格为杂志档案风：纯白背景、黑字、现代无衬线、细字重、少量细线、内容优先。
- 首页排版固定为“精选作品优先 + 作品总览”；分类方式固定按撰稿类型：人物专访、文旅、品牌稿、评论、报道。
- 作品分类区标题为 `写作领域`，分类按钮每行最多三个。
- 作品总览中日期和媒体合并显示，例如 `2026-06-29 男人风尚`；没有作品图片的条目也保留图片占位，使标题与有图条目对齐。
- 精选作品严格以台账 `是否为精选作品` 列为准；不要再用标签或备注中的“精选”字样判断，也不要在没有勾选时自动取最新三篇。
- 精选作品预览显示类似 PDF `精选节选` 的内容，不再简单显示文章第一段。
- 网页使用统一 type scale：正文 `16px`，`h1` 为 `42px`，`h2` 为 `28px`，标题字号不得超过 `44px`；标题与正文的视觉断层要尽量小。
- 单篇文章页保持窄栏长文阅读版，正文行距和段落间距应较宽松。
- 单篇文章页需要识别 Markdown 章节标题，例如 `##`、`###`，并按标题级别设置对应字号。

## 微信公众号正文清洗规则

- 微信公众号优先读取 `js_content` / `rich_media_content` 正文区域。
- 入库时应删除类似图注的非正文内容，只保留成段落的正文。
- 文末 `撰稿`、`撰文`、`作者`、`编辑`、`责编` 等编辑信息需要自动读取并保留，例如 `撰文：Susu`、`编辑：杨叶`；如果这些信息在网页正文中较早出现，入库时也要统一移动到文章最后。
- 应过滤的典型内容包括：
  - 作品图注，例如 `李锋，《盲》，琉璃`。
  - 服装或品牌露出，例如 `外套、针织衫、长裤 均为 Brioni 鞋 Tod's`。
  - 图片说明，例如 `特茹河的日落，以及从船上欣赏里斯本城市风貌`。
  - 图片来源、摄影、供图、Courtesy 等 credit 信息。
- 不要用过宽的短句删除规则，避免误删正常正文。

## PDF 输出规则

- PDF 文件名格式为 `生成日期-SuFANxx类作品集.pdf`，例如 `20260701-SuFAN文旅类作品集.pdf`。
- 全量 PDF 文件名格式为 `生成日期-SuFAN全部作品集.pdf`；当指令包含 `全部`、`全量`、`所有` 或 `完整` 时，导出全部已入库作品。
- PDF 首页标题为 `Su FAN xx类作品集`。
- 全量 PDF 首页标题为 `Su FAN 全部作品集`；当指令包含 `分门别类` 时，目录和正文按撰稿类型分组。
- PDF 首页包含与网页一致的个人简介和合作媒体信息；个人简介分三段展示。
- PDF 首页包含可点击目录，格式为 `媒体名｜标题    页码`；目录页码右对齐，文字左对齐，条目链接到对应正文。
- 首页页码使用罗马数字；正文从下一页开始，页码使用阿拉伯数字。
- PDF 正文每篇作品单独开页，结构为：标题、媒体/日期/类型、阅读原文链接、正文全文；不单独输出 `#标签`、精选节选或 `正文` 提示字样。
- PDF 正文字号关系：文章标题三号字，一级标题小四号字，二级标题和正文五号字。
- 如果作品的 `链接/文件名` 是网址，PDF 正文页在媒体/日期/类型下方显示可点击的 `阅读原文` 超链接；纸刊文件名不生成该链接。

## 用户新增作品时，主动提醒补齐这些信息

如果用户只发链接、文件名、文章标题或说“帮我入库”，先检查台账是否已有完整信息；缺失时提醒用户补齐，不要主观猜测。

### 新媒体文章必填信息

请提醒用户在 `portfolio_registry.xlsx` 的 `作品台账` 新增一行，并填写：

- `标题`：可留空，入库时尽量从网页提取。
- `来源类型`：填 `新媒体`。
- `链接/文件名`：填发布链接。
- `撰稿类型`：例如 `人物专访`、`文旅`、`品牌稿`、`评论`、`报道`。
- `主题标签`：例如 `#人物特稿 #艺术` 或 `文旅、城市更新`。
- `发布媒体`：发布平台或媒体名称。
- `发布日期`：建议格式 `yyyy-mm-dd`。
- `是否公开展示`：填 `是` 或 `否`。
- `是否为精选作品`：填 `是` 或 `否`；首页精选作品优先读取这一列。
- `作品图片`：可选，填写本地图片文件名、本地路径或图片 URL；用于首页精选作品、作品总览和单篇文章页。
- `处理状态`：填 `待处理`。

### 纸刊文章必填信息

请提醒用户先把文档放入 `inbox/`，支持 `.docx`、`.pdf`、`.txt`、`.md`，再在台账新增一行：

- `来源类型`：填 `纸刊`。
- `链接/文件名`：填 `inbox/` 内的文件名，不要填完整路径。
- 其余字段同新媒体文章。
- `处理状态`：填 `待处理`。

## 常用执行方式

用户明确要求执行时，直接在项目根目录运行对应命令，并反馈结果：

```bash
./portfolio.sh ingest
./portfolio.sh site
./portfolio.sh pdf "文旅类撰稿人岗位"
./portfolio.sh pdf "全部作品集，分门别类"
./portfolio.sh all "文旅类撰稿人岗位"
```

执行后需要检查：

- 台账状态是否变成 `已入库`、`需手动补充` 或 `错误`。
- `content/` 是否生成对应 `.md`。
- 更新网站后，`public/index.html` 是否存在。
- 生成 PDF 后，`exports/pdf/` 是否存在对应文件。

## 处理原则

- 不猜文章内容；微信公众号或网页无法提取时，标记为 `需手动补充`。
- 微信公众号正文提取后要清理图注、图片说明和品牌露出等非正文内容。
- Markdown 文件命名使用 `发表日期-刊物-文章名.md`，文章名去掉标点符号，只保留文字和数字，方便用户后续编辑。
- 不擅自改无关文件。
- 能通过脚本完成的，直接执行并验证。
- 如果用户只想知道命令，直接给命令，不额外运行。
- 如果用户要生成某个岗位方向的 PDF，先确认或使用用户给出的岗位/主题关键词作为筛选条件。
