# 自动化作品集整理流程

这套流程以 `portfolio_registry.xlsx` 为唯一录入入口，配合本地文件夹完成文章入库、摘要确认、静态网站更新和岗位定制 PDF 输出。

后续在本项目里开启新对话时，可让助手先读取 `AGENTS.md`。README 面向日常操作，AGENTS 面向助手执行规则。

## 1. 文章入库

所有命令都在项目根目录运行。

### 新媒体文章

先在 `portfolio_registry.xlsx` 的 `作品台账` 新增一行，至少填写：

- `Keyline`：选填，一句话总结文章核心内容、报道方法和体现的个人能力；如果不填，summary 阶段会自动生成。
- `来源类型`：填 `新媒体`。
- `链接/文件名`：填发布链接。
- `撰稿类型`：例如 `人物专访`、`文旅`、`品牌稿`、`评论`、`报道`。
- `主题标签`：多个标签可用中文顿号、逗号或分号分隔。
- `发布媒体`、`发布日期`。
- `是否公开展示`、`是否为精选作品`：填 `是` 或 `否`。
- `处理状态`：填 `待处理`。
- `作品图片`：可选，填 `input/assets/` 中的文件名、本地路径或图片 URL。

`标题` 可以留空，入库时会尽量从网页提取。

### 纸刊文章

先把原始文档放入 `input/inbox/`，支持 `.docx`、`.pdf`、`.txt`、`.md`。然后在台账新增一行：

- `来源类型`：填 `纸刊`。
- `链接/文件名`：填 `input/inbox/` 内的文件名，不要填完整路径。
- 其余字段同新媒体文章；`Keyline` 同样选填。
- `处理状态`：填 `待处理`。

### 运行入库

```bash
./portfolio.sh ingest
```

入库会提取正文、清洗内容、生成 `output/content/` 中的标准化 Markdown，并更新台账状态。新媒体网页正文不足或平台限制时，会标记为 `需手动补充`，不猜测文章内容。

微信公众号正文会优先读取 `js_content` / `rich_media_content`，并过滤图注、服装/品牌露出、图片来源等非正文内容；`撰稿`、`撰文`、`作者`、`编辑`、`责编` 等编辑信息会保留并统一放到文末。

## 2. 摘要生成

入库后不要直接运行一键流程跳过确认。正确流程是：

1. 读取新生成的 `output/content/` 正文。
2. 参考现有 `output/summaries/` 中已确认作品的风格；可让助手使用 `$portfolio-summary-style` 复刻手工摘取风格。
3. 在对话中生成并展示 keyline 与章节精选候选；如果台账已填写 `Keyline`，沿用人工版本并标明“沿用台账 Keyline”；如果台账为空，自动生成并标明“AI 生成 Keyline”。
4. 用户确认或修改后，才写入对应 `output/summaries/` 文件。

summary 文件规则：

- `keyline` 只放在 YAML 字段中，简洁、有信息密度，不加“概述”“文章简介”等前缀；人工已填时不覆盖，未填时由 summary 阶段生成。
- 正文只放章节精选，不重复 keyline。
- 如果原文已有章节结构，保留原章节结构，不自行新增 `## 精选节选` 或创造新分段。
- 如果原文没有章节结构，可以使用 `## 精选节选`。
- 未明确确认状态时，`summary_status` 保持 `待审核`；只有 `summary_status: 已确认` 的内容会进入网站和摘要版 PDF。

如需检查或补齐缺失的摘要草稿，可以运行：

```bash
./portfolio.sh summaries
```

新增文章录入时，优先在对话中完成上述 AI 确认流程，不要用 `./portfolio.sh summaries` 静默生成并写入未确认内容。

## 3. 检查入库与摘要情况

入库后重点检查：

- 台账 `处理状态` 是否变成 `已入库`、`需手动补充` 或 `错误`。
- `output/content/` 是否生成对应 `.md` 正文归档。
- 新文章是否等待用户确认 summary，而不是静默写入 `output/summaries/`。
- summary 文件的 `summary_status` 是否为预期状态。

常见状态含义：

- `已入库`：正文已提取并归档。
- `需手动补充`：网页正文过短、平台限制或文件内容不足，需要手动补正文。
- `错误`：通常是缺少来源类型、链接/文件名，或纸刊文件无法读取。

入库后的 Markdown 文件名格式为 `发表日期-刊物-文章名.md`，文章名会去掉标点符号，便于后续查找和编辑。

## 4. 网站更新

网站只展示 `是否公开展示` 为 `是` 且 `处理状态` 为 `已入库` 的作品。单篇文章页只读取 `output/summaries/` 中 `summary_status: 已确认` 的 keyline 与章节精选；未确认时显示基础信息、阅读原文和“摘要待确认”，不展示 `output/content/` 全文。

更新静态网站：

```bash
./portfolio.sh site
```

更新后检查：

- `output/site/index.html` 是否存在。
- 首页、作品总览、单篇文章页、CSS 和图片资源是否能正常打开。

更新网站并备份到 GitHub：

```bash
./portfolio.sh publish "更新作品集网站"
```

`publish` 会先重新生成 `output/site/`，再提交到 Git，并在配置了 GitHub 远程仓库后推送。GitHub 只作为代码和网站产物备份，不负责自动上线。

发布 Vercel 全球版：

```bash
./portfolio.sh deploy-vercel
```

发布腾讯云服务器国内版：

```bash
./portfolio.sh deploy-tencent-server
```

公开产物边界：

- 网站只发布 `output/site/`，不发布 `portfolio_registry.xlsx`、`.env.tencent-server`、`.vercel/` 或 `output/content/`。
- 联系方式页的 PDF 下载只使用最新 `SuFAN全部作品集-摘要版.pdf`。
- 全文版 PDF 不默认复制到网站产物。

## 5. PDF 作品集更新

默认生成摘要版 PDF，只读取已确认 summary；只有明确使用 `pdf-full` 或查询中包含 `全文版` 时，才读取 `output/content/` 生成全文版。

生成摘要版 PDF，例如文旅类岗位：

```bash
./portfolio.sh pdf "文旅类撰稿人岗位"
```

生成摘要版全部作品集，并按撰稿类型分门别类：

```bash
./portfolio.sh pdf "全部作品集，分门别类"
```

生成全文版 PDF：

```bash
./portfolio.sh pdf-full "全部作品集，分门别类"
```

生成后检查 `output/pdf/` 是否出现对应文件。

当指令包含 `全部`、`全量`、`所有` 或 `完整` 时，导出全部已入库作品；当指令包含 `分门别类` 时，目录和正文按撰稿类型分组。PDF 正文页会在媒体/日期/类型下方显示可点击的 `阅读原文` 超链接，纸刊文件名不生成该链接。

已有 summary 已确认的日常更新，可以一键完成入库、网站更新和 PDF：

```bash
./portfolio.sh all "文旅类撰稿人岗位"
```

新增文章录入时不要使用 `all`，避免跳过 AI 确认流程。

## 文件夹说明

- `portfolio_registry.xlsx`：作品总台账，填写链接、文件名、撰稿类型、发布媒体、图片路径等信息。
- `input/assets/`：放首页照片、作品配图和联系方式图标；首页个人照片固定命名为 `profile.jpg`。
- `input/inbox/`：放纸刊原始文档。
- `input/contact_info_template.md`：联系方式页面内容。
- `output/content/`：自动整理后的标准化作品正文，不公开到网站。
- `output/summaries/`：用于网站和默认 PDF 的 keyline 与章节精选。
- `output/site/`：生成的静态网站文件。
- `output/pdf/`：生成的 PDF 作品集。
- `work/tmp/`：临时文件、预览图和中间草稿。
- `scripts/portfolio.py`：主操作入口。

## 台账字段

台账固定使用这些列：

`作品ID`、`标题`、`Keyline`、`来源类型`、`链接/文件名`、`撰稿类型`、`主题标签`、`发布媒体`、`发布日期`、`是否公开展示`、`是否为精选作品`、`处理状态`、`备注`、`作品图片`

填写和格式规则：

- `作品ID` 可留空，入库时会自动生成。
- `Keyline` 选填；填写后 summary 阶段沿用，不填写则由 summary 阶段自动生成。
- `来源类型` 填 `新媒体` 或 `纸刊`。
- `是否为精选作品` 严格控制首页精选作品；不会根据标签或备注自动判断精选，也不会在未勾选时自动取最新三篇。
- `作品图片` 可填文件名、本地路径或图片 URL；只填文件名时优先从 `input/assets/` 查找。
- `作品台账` 第一行冻结。
- `来源类型`、`撰稿类型`、`是否公开展示`、`是否为精选作品`、`处理状态` 使用下拉列表。
- 表格边框只使用横线：正文行使用灰色横线，第一行使用上下黑色边框，不设置竖线和表格尾边框。

## 网站展示原则

- 首页标题固定为 `Su FAN 作品集`。
- 首页个人简介、合作媒体汇总和 `input/assets/profile.jpg` 个人照片保持与当前网站一致。
- 当前网站风格为杂志档案风：纯白背景、黑字、现代无衬线、细字重、少量细线、内容优先。
- 首页排版为“精选作品优先 + 作品总览”；分类方式按撰稿类型，作品分类区标题为 `写作领域`。
- 作品图片保持彩色显示，主体完整，不裁切关键信息；未填写图片时按当前版式处理占位与对齐。
- 单篇文章页保留 `Published Work`、阅读原文、keyline、分割线和章节精选结构，并保持窄栏长文阅读版。

## 发布与安全说明

首次使用腾讯云服务器部署前，复制示例配置：

```bash
cp .env.tencent-server.example .env.tencent-server
```

然后填写服务器公网 IP、SSH 用户名、端口、远程目录和可选 SSH 私钥路径。`.env.tencent-server` 只保存在本机，已被 Git 忽略；不要把真实服务器地址、私钥或备案号写入示例文件。

SSH 私钥不要放在项目文件夹内，建议放在 `~/.ssh/`，并在 `.env.tencent-server` 中只填写路径。自动部署默认禁用密码 SSH 和交互式登录，只允许公钥认证；如果失败，先检查服务器公钥和安全组，不要改成密码部署。

正式访问入口统一使用无 `www` 主域名：

```text
https://sufan-freelancewriter.com.cn
```

Vercel 仍保留为全球部署来源；`sufan-portfolio.vercel.app` 只作为部署调试或回滚检查使用，不作为对外访问入口。
腾讯云国内版保留为支持路径，备案完成后应使用正式域名和 HTTPS，不要长期把公网 IP 当作正式入口。
